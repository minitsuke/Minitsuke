# Contributing to Minitsuke

## Principles

- Keep changes small and clear.
- Document decisions.
- Follow naming conventions.
- Do not hardcode learning content.
- Prefer reusable components and modular features.

## Branch Naming

```text
feature/<short-name>
fix/<short-name>
docs/<short-name>
```

## Commit Style

```text
docs: add project bible
feature: add lesson card
fix: correct grammar metadata
```
